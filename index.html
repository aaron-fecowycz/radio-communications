Standard Terminology

The use of standard phrases will ensure exact meaning a sender wishes to convey is given to the receiever and no time is wasted.

Affirmative
Indicates a positive response
Negative
Indicates a negaitive response
All after / All before
Used to identify/clarify parts of a message
Check

Only to be used when a lengthy message is being transmitted.

The sender wil break tranismission periodically using the word "Check" ; this will enable the recipient to check that part of the mssage so far received. If the message has been receievd and understood the recipient will reply "Check" and the transmission will continue.

Closing Down
Used when a mobile ceases operation and teh crew are to be contacted by a different communications device.
Correction
Cancel the last word or phrase sent, or cancel word or indicated and substitue with the correct information. Also used to correct information that has been sent out but on repeating back (ie check) is apparent that it's been misheard.
Fire Flash
Used by control for transmission of urgent operational messages whilst at stage 1 (loss of individual MDT) or stage 2 (total loss of data functions) fall back procedures are in operation.
Priority message
Used by crews when it is deemed necessary to transmit urgent operational message. This will indicate to control that priority must be given to the message in prefference to others.
I Spell
Used when spelling a word phonetically, abbreviation letter or series of letters.
No Play

Used during exercise to qualify the following information being real and not part of the exercise.

Example: "No Play, No Play, No Play. Firefighter injured assistance required".

Roger
Indicates that a message has been receved and understood.
Wilco
Indicates that a message requiring action has been receieved and understood and will be complied with.
Over
Control and mobiles use of the term "Over" after each radio transmission. "Over" indicates that the transmission is over.
Out
The transmission is ended and I do not expect you to make a further transmission on this subject. I am listening to messages from other mobiles. Out will only be used by Control at the end of a communication.
Repeat
To verfy text of a message or of part indicated with the originator.
Wait
A pause of a few seconds will follow, other users must not intterupt.
Wait out
I hav finished for the time being and will call you again later. Other users may transmit if neccessary.
Kilo One
The term "Kilo One" will be used on the inicident ground and in standard radio messages to identify a deceased casualty.

Message Log

Mobile to Incicent Message

It is not sufficient just to book 'mobile' unless this is purely via MDT and no voice message is being passed. The exact address must be clearly indicated by the appliance at this stage, to ensure the correct address has been received and understood.

In addition the rank and name the Officer in Charge of the appliance together with the number of additional riders must be given, unless this information has been entered into the MDT.

First Impression Message

A first impressions message is intended to give fire control and oncomming appliances a brief initial apprasial of the nature and scale of the incident. it does not replace an "Informative Message". A tactical mode should only be included where a DRA has been conducted.

On arrival at an incident where it is identified that persons are trapped or unaccounted for, the message "Persons reported" should be passsed to fire control immediately as either part of the first impression message r as a seperate Informative Message:

"this incident is now persons reported."

Assistance Message

An assistance message is one which requests any equipment, information, appliance or service not readily available on the incident ground. A specific reason for the assistance message allows Fire Control to assign the correct resource to the incident.

All Assistance messages must be voice communicated and are high priority messages and should be actioned without delay.

Establishing a Command Point

As an incident progresses the Incident Commander may deem it necessary to establish a Command Point.

This will be singular point which will manage communications, the flow of information from and to the incident ground, requests for resources and the collation of information e.g.decision logs, site maps, personnel in attendance etc.

Once the decision to establish a Command Point has been taken Fire Control must be informed of which appliance has been designated, the name of the Command Point and the name of the Incident Commander. This information should be passed as an informative message.

As a general rule, a Command Point should be established at all incidents of two appliances and above.

Once a command point has been named then subsequent initial contact with Fire Control must be made using the name of the Command Point. The identity of the appliance will already be attached as the Command Point in Fire Control. During the passage of messages whoever is in overall charge at the incident should be referred to as 'Incident Commander' as their name will be attached to the Incident Commander role. Any change in Incident Commander should be communicated to Fire Control.

Closing Down a Command Point

As an incident is being downsized and appliances start to depart the Command Point can revert back to an appliance callsign. This should be sent as an informative message and contain the appliance call sign which will now pass and receive messages and the current Incident commander.

Stop Message

A "Stop message" from an incident informs control the IC has sufficient resource on the incident ground to deal with the incident and must always be sent via a voice message.

Within a "Stop message" content there must also be given an indication of how long the appliances are likely to be detained at the incident.
